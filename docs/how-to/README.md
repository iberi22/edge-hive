## What are How-To Guides?

How-to guides are **problem-solving recipes**. They show you how to accomplish specific goals with Git-Core Protocol.

### When to use how-to guides

- ‚úÖ You know what you want to achieve
- ‚úÖ You need step-by-step directions
- ‚úÖ You're working on a real task
- ‚úÖ You want the shortest path to your goal

### What how-to guides are NOT

- ‚ùå Learning lessons (see [Tutorials](../tutorials/))
- ‚ùå Complete reference (see [Reference](../reference/))
- ‚ùå Background theory (see [Explanation](../explanation/))

## How-To Principles (Di√°taxis)

Our how-to guides follow these patterns:

| Principle | Meaning |
|-----------|---------|
| **Goal-oriented** | Solve a specific problem |
| **Practical** | Actionable steps, not theory |
| **Flexible** | Adaptable to your situation |
| **Focused** | Only what you need, nothing extra |
| **Results-driven** | You get something done |

### Expected Experience

When following a how-to guide, you should:

- ‚úÖ Know exactly what problem you're solving
- ‚úÖ See clear, actionable steps
- ‚úÖ Reach your goal quickly
- ‚úÖ Be able to adapt steps to your needs

## Guide Format

Each how-to guide follows this structure:

```markdown
# How to {Accomplish Goal}

## Problem
Brief description of what you're trying to achieve

## Solution
Step-by-step instructions

## What You Need
Prerequisites or requirements

## Steps
1. Clear action
2. Expected result
3. Next action

## Troubleshooting
Common issues and solutions

## See Also
- Links to related guides
- Links to reference docs
```
