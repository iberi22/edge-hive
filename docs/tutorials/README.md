## Tutorial Principles (Di√°taxis)

Our tutorials follow strict pedagogical principles:

| Principle | Meaning |
|-----------|---------|
| **Learning-oriented** | We're here to learn, not to get work done |
| **Action-focused** | You *do* things, not just read about them |
| **Concrete** | Real examples, not abstract concepts |
| **Successful** | Every step works and produces visible results |
| **Guided** | Clear directions, no guessing |
| **Minimal explanation** | Links to [Explanation](../explanation/) for deeper understanding |

### Expected Experience

When following a tutorial, you should feel:

- ‚úÖ Guided and supported
- ‚úÖ Confident at each step
- ‚úÖ Proud of what you've built
- ‚úÖ Eager to learn more
